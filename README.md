# Kronos-R：基于离散语言建模的金融时序基础模型

## 项目研究目的

金融时序预测面临低信噪比、非平稳性等固有挑战，传统方法在处理长程依赖时力不从心。语言模型范式（将连续 K 线离散化为 token 后做 next-token prediction）为金融时序建模提供了新思路，但仍有以下关键问题亟待解决：

1. **码本坍缩**——VQ-VAE 分词器死码率高，模型"词汇量"受限。
2. **方向感知缺失**——CE 损失不区分涨跌，方向准确率长期徘徊于随机水平。
3. **多步误差累积**——训练时 teacher-forcing 与推理时自回归生成存在分布不匹配。
4. **泛化能力不明**——后训练策略可能过度拟合特定股票模式，跨股票泛化能力需验证。
5. **算力约束**——如何在单 GPU（8 GB VRAM）条件下实现高效训练。

## 主要研究内容

**1. 金融时序离散化分词方法**。研究 BSQ 隐式码本分词器，从数学上消除死码问题；设计两级层次化量化结构（粗粒度+细粒度），平衡重建精度与 token 多样性。

**2. 因果自回归 Transformer 基础模型**。融合线性注意力与多尺度局部注意力，设计 Latent Reasoner 实现并行推理；探索 RoPE、ALiBi 等位置编码策略，构建百万级参数的高效时序模型。

**3. Tokenizer 与 Base Model 的联合影响机制**。系统分析码本利用率、token 多样性、学习率调度、模型容量等变量与方向准确率的关联，突破 DA 在 50% 附近的瓶颈。

**4. 方向感知后训练**。研究 ExPO 偏好优化方法，通过 winner/loser 候选对比训练提升方向判别能力，同时保持数值预测精度不退化。

**5. 多步自回归 Rollout 后训练**。探索 Oracle 引导的步骤级筛选与 Expert Iteration 轨迹级筛选两种路线，结合课程学习和 KL 正则化，降低多步预测的路径 MAPE。

**6. 后训练策略泛化能力评估**。在验证集和留出 demo 集上系统对比各方法的泛化表现，回答步骤级筛选与轨迹级筛选孰优孰劣的核心问题。

**7. 高效训练工程优化**。实现梯度检查点、混合精度训练、mmap 大数据加载、自动 batch 调优等优化，建立消费级 GPU 上一键化训练流水线。
